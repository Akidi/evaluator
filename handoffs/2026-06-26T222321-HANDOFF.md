## State of `evaluator/evaluator.ts` — now feature-complete

All six `Node` union members (`Num`, `Ident`, `Unary`, `Binary`, `Call`, `Ternary`) are implemented and tested. 93 tests passing across the whole project (50 in `evaluator/evaluator.test.ts`).

This session's additions, in order:
1. **Comparison ops** (`EQ`, `NEQ`, `LT`, `LTE`, `GT`, `GTE`) wired into `COMP_OPS`, mirroring the `MATH_OPS` pattern.
2. **Remaining arithmetic ops** (`MINUS`, `SLASH`, `PERCENT`, `CARROT`) added to `MATH_OPS`.
3. **Real-boolean `NOT` test** — previously blocked because no AST path produced a real boolean without `Binary` comparisons; now exists (`NOT` over a `Binary EQ` result).
4. **`AND`/`OR` short-circuit** — special-cased branches in `evaluate()`'s `Binary` case, evaluated *before* the `MATH_OPS`/`COMP_OPS` table lookup. Evaluates `left` first, type-checks boolean, short-circuits without touching `right` when the result is already determined (`AND` on `false`, `OR` on `true`). Verified by tests that pass an `Ident` to an undefined variable as the unevaluated branch — if it were evaluated, the test would throw `UndefinedVariableError` instead of returning the short-circuited value.
5. **`Ternary`** — `test`/`ifTrue`/`ifFalse`, type-checks `test` is boolean (`InvalidOperandError` otherwise), evaluates only the chosen branch (verified the same way as AND/OR, via an unevaluated `Ident` to an undefined var on the untaken branch).
6. **`Call`** — looks up `ast.callee.name` in `fns`, throws `UndefinedFunctionError` if missing, otherwise evaluates `args` and invokes.
7. **Arity validation** (the one-time Claude-implemented exception) — `defineFn(name, fn, arity)` is now a required 3-arg signature; `FnEnv` is `Map<string, { fn: EvalFn; arity: number }>`; `Call` throws the new `ArityMismatchError` (added to `evaluator/errors.ts`) when `ast.args.length !== entry.arity`.
8. **Full-pipeline stress test** added (`evaluator.test.ts`, last test) — runs a real formula string through `Lexer.scan` → `Parser.parse` → `Evaluator.evaluate` end to end, using custom vars (`a=3, b=5, c=2`) and two custom functions (`avg`, `clamp`) deliberately *not* mirrored in `MATH_OPS`/`COMP_OPS`, to prove `Call` integrates with real user-defined logic rather than just table-driven ops. Formula: `"avg(a, b) + clamp(10, 0, c) * b - a ^ c == 7 ? !(a < b) || c >= b : a != b && b > c"` → evaluates to `true` (math reduces to `5 == 7` → false → ifFalse branch `(3!=5) && (5>2)` → `true`).
