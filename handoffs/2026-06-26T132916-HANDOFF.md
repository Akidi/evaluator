## Decisions made this session

- `evaluator/types.ts` finalized:
  ```ts
  export type EvalFn = (...args: unknown[]) => number | boolean;
  export type VarEnv = Map<string, number>;
  export type FnEnv  = Map<string, EvalFn>;
  ```
  - `unknown[]` args chosen over `any[]` deliberately ‚Äî forces narrowing/casting at call sites instead of silently allowing anything.
  - Return type tightened from `unknown` to `number | boolean` since that's all `evaluate()` ever produces.
- **`VarEnv`/`FnEnv` are evaluator instance state, not `evaluate()` call params.** `IEvaluator` should grow `setVar(name, value)` and `defineFn(name, fn)` methods so vars get assigned before/between evaluations and users can register custom functions into `FnEnv` directly ‚Äî confirmed this is "free" given `EvalFn`'s loose signature (no arity/shape opinion baked into the type).
- `evaluate(ast: Node)` confirmed correct ‚Äî `Node` is the parser's actual AST type (`parser/types.ts`). **Flagged but not fixed: `evaluator/types.ts` is missing `import { Node } from "../parser/types"`.**
