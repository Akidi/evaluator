## Open questions (not yet decided)

- **Are chained comparisons (`a < b < c`) legal, and if so what do they mean?** Math reads this as `a < b AND b < c` (a true chain). Most C-like languages instead parse it left-to-right through the normal bp loop as `(a < b) < c` — comparing a boolean-ish result against `c`, which is usually nonsensical/a type error. Needs a decision before comparisons (bp 4) are implemented: either (a) make comparison ops non-associative — parsing two in a row without an explicit `AND` is a parse error, or (b) special-case chained comparisons to expand into the math meaning, or (c) allow the C-like left-fold behavior and let it be a runtime/type error later. Not yet decided.
